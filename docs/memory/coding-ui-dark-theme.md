# UI dark 主题独立设计

conduct UI 的 dark 主题必须作为独立视觉系统设计，不能通过反转 light 色值完成。

- 页面底层、普通表面、抬升表面、hover 表面分别使用语义令牌，靠层级、边框和克制的阴影表达深度。
- 主按钮、编辑器栏、工具提示分别拥有独立的前景和背景令牌；禁止复用 `--ink` 等正文颜色作为组件背景。
- 成功、运行、警告、失败等语义色在两个主题中保持含义一致，只调整明度和饱和度以适配背景。
- 主题切换不得改变布局和组件尺寸。小字号文字与实际背景的对比度至少为 `4.5:1`，用自动测试守住关键组合。
- light 是既有视觉基线；改进 dark 时，新增令牌在 light 中应映射回原有值，避免无意改变 light 外观。
- 主题持久化以 `~/.conduct/settings.json` 的 `theme` 为唯一真值；属性缺失表示跟随系统，浏览器不得用 `localStorage` 维护第二份偏好。

**Why:** 简单反转或机械复用 light 色值会破坏 dark 主题的表面层级、语义色和文字对比度，导致视觉质量与可读性下降。
**How to apply:** 新增或调整 UI 颜色、组件状态、主题切换及主题持久化时，按本规则设计令牌并补关键对比度与持久化测试。
